> **版本**：v0.1（通信链路验证阶段）
> **适用对象**：眼镜端开发者
> **最后更新**：2026-03-30

---

## 1. 系统概述

本阶段验证以下通信主链路：

```
眼镜端拍照
    ↓  HTTP POST (image/jpeg)
手机端接收图片 → 模拟识别 → 返回 JSON
    ↓
眼镜端解析并显示结果
```

**网络拓扑**：手机开启个人热点，眼镜连接该热点，双方处于同一局域网，无需公网。

## 2. 网络环境配置

| 项目 | 说明 |
|------|------|
| 热点开启方 | 手机（使用者手机） |
| 连入方 | 眼镜 |

## 3. API 接口规范

### 3.1 接口总览

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/analyzeDrug` | 上传药物图片，获取识别结果 |

### 3.2 POST /analyzeDrug

#### 请求

| 属性 | 值 |
|------|-----|
| URL | `http://192.168.43.1:8080/analyzeDrug` |
| Method | `POST` |
| Content-Type | `image/jpeg` |
| Body | JPEG 图片原始二进制数据（raw binary） |

**重要**：Body 直接传图片字节流，**不要使用 multipart/form-data 或 base64**。

#### 响应

**HTTP 200 — 成功**

```
Content-Type: application/json
```

```json
{
  "success": true,
  "drugName": "测试药物",
  "confidence": 0.95,
  "warningText": "这是手机端返回的测试结果",
  "needConfirm": true
}
```

**字段说明**

| 字段 | 类型 | 说明 |
|------|------|------|
| `success` | Boolean | `true` 表示处理成功，`false` 表示服务器内部出错 |
| `drugName` | String | 识别出的药物名称（当前阶段固定返回"测试药物"） |
| `confidence` | Double | 识别置信度，范围 0.0 ~ 1.0 |
| `warningText` | String | 给用户的提示文字，直接展示在眼镜 AR 界面 |
| `needConfirm` | Boolean | `true` 时需要用户在眼镜端进行二次确认操作 |

**HTTP 500 — 服务器错误**

```json
{
  "success": false,
  "error": "服务器内部错误: <错误描述>"
}
```

**HTTP 404 — 未知接口**

```json
{
  "success": false,
  "error": "未知接口: /wrongPath"
}
```

## 4. 眼镜端对接示例

### 4.1 伪代码流程

```
1. 触发拍照（用户手势 / 语音指令 / 自动检测）
2. 获取 JPEG 图片字节数组
3. HTTP POST 发送至 http://192.168.43.1:8080/analyzeDrug
   - Header: Content-Type: image/jpeg
   - Header: Content-Length: <图片字节数>
   - Body: <图片字节数组>
4. 接收响应 JSON，解析字段
5. 在 AR 界面叠加显示 drugName + warningText
6. 若 needConfirm == true，显示确认按钮等待用户操作
```

### 4.2 Android（Kotlin）示例 — OkHttp

如眼镜端也是 Android 系统，可使用 OkHttp：

```kotlin
import okhttp3.MediaType.Companion.toMediaType
import okhttp3.OkHttpClient
import okhttp3.Request
import okhttp3.RequestBody.Companion.toRequestBody
import org.json.JSONObject
import java.util.concurrent.TimeUnit

val client = OkHttpClient.Builder()
    .connectTimeout(5, TimeUnit.SECONDS)
    .readTimeout(10, TimeUnit.SECONDS)
    .build()

fun analyzeDrug(jpegBytes: ByteArray): DrugResult? {
    val body = jpegBytes.toRequestBody("image/jpeg".toMediaType())
    val request = Request.Builder()
        .url("http://192.168.43.1:8080/analyzeDrug")
        .post(body)
        .build()

    return try {
        val response = client.newCall(request).execute()
        if (response.isSuccessful) {
            val json = JSONObject(response.body!!.string())
            DrugResult(
                drugName    = json.getString("drugName"),
                confidence  = json.getDouble("confidence"),
                warningText = json.getString("warningText"),
                needConfirm = json.getBoolean("needConfirm")
            )
        } else null
    } catch (e: Exception) {
        null  // 网络异常处理：可弹出"无法连接手机"提示
    }
}

data class DrugResult(
    val drugName: String,
    val confidence: Double,
    val warningText: String,
    val needConfirm: Boolean
)
```

### 4.3 Java（HttpURLConnection）示例

如眼镜端 SDK 基于 Java：

```java
public static String analyzeDrug(byte[] jpegBytes) throws Exception {
    URL url = new URL("http://192.168.43.1:8080/analyzeDrug");
    HttpURLConnection conn = (HttpURLConnection) url.openConnection();
    conn.setRequestMethod("POST");
    conn.setDoOutput(true);
    conn.setRequestProperty("Content-Type", "image/jpeg");
    conn.setRequestProperty("Content-Length", String.valueOf(jpegBytes.length));
    conn.setConnectTimeout(5000);
    conn.setReadTimeout(10000);

    try (OutputStream os = conn.getOutputStream()) {
        os.write(jpegBytes);
    }

    int code = conn.getResponseCode();
    if (code == 200) {
        try (InputStream is = conn.getInputStream()) {
            // 读取响应 JSON 字符串并解析
            return new String(is.readAllBytes(), StandardCharsets.UTF_8);
        }
    }
    return null;
}
```

## 5. 错误处理建议

| 错误场景 | 原因 | 眼镜端建议处理 |
|----------|------|----------------|
| 连接超时 | 手机未开热点 / 服务未启动 / IP 错误 | 显示"无法连接手机，请确认热点已开启" |
| HTTP 500 | 手机端处理图片出错（如磁盘满） | 显示"识别失败，请重试" |
| `success: false` | 服务端逻辑错误 | 读取 `error` 字段内容上报日志 |
| 响应解析失败 | 网络中断导致响应不完整 | 重试一次，仍失败则提示用户 |

## 6. 调试方法

### 6.1 查看手机端日志

手机连接 PC 后执行：

```bash
adb logcat -s LocalHttpServer HttpServerService
```

关键日志标签：
- `LocalHttpServer` — 请求到达、图片大小、保存路径、返回结果
- `HttpServerService` — 服务启动/停止状态

### 6.2 取出手机保存的图片

每次请求的图片都会保存到手机缓存目录，可拉取验证：

```bash
adb pull /data/data/com.example.test/cache/drug_images/
```

### 6.3 确认服务运行

```bash
# 检查 8080 端口是否在监听
adb shell "ss -tlnp | grep 8080"
```

## 8. 常见问题

**Q：手机热点 IP 不是 192.168.43.1 怎么办？**
A：在手机 App 主界面查看"本机 IP"字段，将眼镜端代码中的 IP 替换为该值。

**Q：能否不用热点，直接连同一 WiFi 路由器？**
A：可以。只要眼镜和手机在同一局域网下，使用手机在 App 中显示的 IP 即可，端口仍为 8080。

**Q：图片格式必须是 JPEG 吗？**
A：当前 Demo 阶段手机端不做格式校验，但建议发送 JPEG 以减小图片体积，降低传输延迟。正式接入模型后需根据模型输入要求处理。

**Q：接口支持并发请求吗？**
A：NanoHTTPD 默认为每个请求分配独立线程，支持并发。Demo 阶段单眼镜单请求，无并发压力。
