我正在开发一个基于 RayNeo X3 Pro AR 眼镜的智能用药辅助系统。当前阶段我只想做一个最小可运行 Demo，用来验证“眼镜端拍照 -> 手机端接收图片 -> 手机端返回识别结果 -> 眼镜端显示结果”这条主链路。

请你使用 Kotlin 帮我生成一个 Android 手机端项目中的核心代码，要求如下：

【项目背景】
- 系统采用三端协同架构：眼镜端、使用者手机端、监护人手机端。
- 当前只实现眼镜端和使用者手机端之间的最小通信闭环。
- 眼镜端负责拍照和显示结果。
- 手机端负责接收图片、做基础处理，并返回一个模拟识别结果。
- 现阶段不接入真实 AI 识别模型，不接入数据库，不接入监护人端。
- 当前重点是先打通通信链路。

【通信方案】
- 使用 HTTP。
- 手机端作为本地 HTTP 服务端。
- 假设手机会开启热点，眼镜连接到手机热点，因此眼镜和手机处于同一局域网中。
- 不考虑公网访问，不考虑云端转发。
- 手机端监听固定端口，例如 8080。
- 提供一个 POST 接口，例如 /analyzeDrug。
- 接口接收 JPEG 图片二进制数据。
- 接口返回 JSON 结果。

【返回 JSON 格式要求】
返回结构示例：
{
  "success": true,
  "drugName": "测试药物",
  "confidence": 0.95,
  "warningText": "这是手机端返回的测试结果",
  "needConfirm": true
}

请为这个 JSON 定义 Kotlin data class。

【功能要求】
1. 在 Android 手机上启动一个轻量本地 HTTP 服务。
2. 提供一个 POST /analyzeDrug 接口。
3. 接收上传的图片数据（image/jpeg）。
4. 将图片保存到手机本地缓存目录，便于调试。
5. 打印日志，包括：
   - 请求到达
   - 图片大小
   - 保存路径
6. 返回固定的模拟 JSON 结果。
7. 代码要清晰，适合作为后续接入真实识别算法的基础。

【技术要求】
- 使用 Kotlin。
- 优先选择简单、轻量、适合本地嵌入 Android App 的方案。
- 如果你选择 Ktor / NanoHTTPD / 其他轻量方案，请说明理由，并给出对应依赖。
- 代码应包括：
  1. Gradle 依赖
  2. Manifest 中可能需要的权限
  3. 启动服务的类
  4. HTTP 路由处理代码
  5. JSON 返回的数据类
  6. 一个简单的 Activity，用来启动和停止本地服务，并显示当前服务状态和本机 IP 地址
- 请尽量避免过度复杂架构。
- 先不要使用 Room、Hilt、Compose 多模块等复杂结构。
- 但代码要有基本可维护性。

【输出要求】
请直接输出完整代码，并按文件划分，例如：
- build.gradle.kts
- AndroidManifest.xml
- MainActivity.kt
- LocalHttpServer.kt
- DrugAnalyzeResponse.kt
如果代码较长，请分块输出，但保证能拼接成一个最小可运行项目。

【额外要求】
- 代码中的注释请写清楚，尤其说明：
  - 手机端为什么要作为 HTTP 服务端
  - 图片是如何接收和保存的
  - 后续如果要接入真实药物识别模型，应该在哪一层扩展
- 默认以 Android 原生开发为准。
- 不要生成伪代码，要生成尽量可运行的 Kotlin 代码。
