# 眼镜端开发指南

> **项目**：RayNeo X3 Pro 智能用药辅助系统 — 最小可运行 Demo
> **版本**：v0.1（通信链路验证阶段）
> **面向读者**：负责实现眼镜端代码的开发者（Claude）
> **最后更新**：2026-03-31

---

## 1. 任务概述

你需要编写 **RayNeo X3 Pro 眼镜端** 的 Android 应用代码，实现以下闭环：

```
用户触发 → 眼镜拍照 → HTTP POST 发送图片 → 手机返回 JSON → 眼镜 AR 界面显示结果
```

手机端代码**已完成**，你只需实现眼镜端。本文档描述了双端通信的全部约定，以及眼镜端需要实现的功能。

---

## 2. 系统架构与分工

```
┌─────────────────────────────────┐     HTTP (局域网)     ┌──────────────────────────────────┐
│         眼镜端（你来写）          │ ───────────────────► │        手机端（已完成）             │
│  RayNeo X3 Pro                  │                       │  Android App (com.example.test)   │
│                                 │ ◄─────────────────── │                                   │
│  - 触发拍照                      │      JSON 响应        │  - LocalHttpServer (NanoHTTPD)    │
│  - 发送图片                      │                       │  - HttpServerService (前台服务)    │
│  - 显示 AR 结果                  │                       │  - MainActivity (Compose UI)      │
└─────────────────────────────────┘                       └──────────────────────────────────┘
```

**手机端角色**：HTTP 服务端，监听 8080 端口，接收图片并返回识别结果。
**眼镜端角色**：HTTP 客户端，拍照后主动发起请求，接收结果后在 AR 界面叠加显示。

## 3. 网络环境

| 项目 | 值 |
|------|----|
| 热点开启方 | 手机（Android 个人热点） |
| 热点连入方 | 眼镜 |
| 手机热点固定 IP | `192.168.43.1`（Android 热点标准地址） |
| 服务端口 | `8080` |
| 协议 | HTTP（明文，无需 TLS） |

**连接示意**：

```
手机热点 (192.168.43.1:8080)
        ↑
        │ WiFi
        │
眼镜 (DHCP 分配地址, 如 192.168.43.xxx)
```

> 如手机未开热点而是连接路由器 WiFi，手机 IP 会变化，需从手机 App 主界面读取"本机 IP"字段。Demo 阶段默认使用热点场景，IP 写死 `192.168.43.1`。

## 4. 通信协议规范

### 4.1 接口一览

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/analyzeDrug` | 上传药物图片，获取识别结果 |

### 4.2 请求格式

```
POST http://192.168.43.1:8080/analyzeDrug
Content-Type: image/jpeg
Content-Length: <图片字节数>

<JPEG 图片原始二进制数据>
```

**关键约束**：
- Body 是 JPEG 图片的**原始字节流（raw binary）**
- **不要**使用 `multipart/form-data`
- **不要**使用 base64 编码
- Header 中必须有 `Content-Type: image/jpeg`

### 4.3 响应格式

**成功（HTTP 200）**

```json
{
  "success": true,
  "drugName": "测试药物",
  "confidence": 0.95,
  "warningText": "这是手机端返回的测试结果",
  "needConfirm": true
}
```

| 字段 | 类型 | 用途 |
|------|------|------|
| `success` | Boolean | `true` = 处理成功 |
| `drugName` | String | 识别出的药物名称，显示在 AR 界面 |
| `confidence` | Double | 置信度 0.0~1.0，可选择性显示 |
| `warningText` | String | 警告/提示文字，直接展示给用户 |
| `needConfirm` | Boolean | `true` 时需要用户二次确认 |

**服务器错误（HTTP 500）**

```json
{
  "success": false,
  "error": "服务器内部错误: <描述>"
}
```

**未知接口（HTTP 404）**

```json
{
  "success": false,
  "error": "未知接口: /wrongPath"
}
```

## 5. 眼镜端需要实现的功能

### 5.1 功能列表（最小 Demo）

1. **触发拍照**：支持至少一种触发方式（建议：点击屏幕按钮，或侧键物理按键）
2. **获取 JPEG 图片字节数组**：调用 RayNeo Camera API 拍照，转为 `ByteArray`
3. **HTTP 请求发送**：在子线程/协程中发送 POST 请求，不阻塞主线程
4. **响应解析**：解析 JSON，提取 `drugName`、`warningText`、`needConfirm`
5. **AR 界面展示**：将结果叠加显示在摄像头预览画面上
6. **连接失败提示**：超时或网络错误时显示友好提示

### 5.2 不需要实现（本阶段超出范围）

- 真实药物识别算法（由手机端占位返回模拟数据）
- 监护人端通知
- 本地数据库存储
- 用户账号登录

## 6. 开发平台说明

### 6.1 RayNeo X3 Pro 基本信息

| 项目 | 值 |
|------|----|
| 操作系统 | Android（AOSP 定制） |
| 开发语言 | Kotlin / Java（标准 Android 开发） |
| 摄像头 API | RayNeo SDK 提供，或标准 Android Camera2 API |
| AR 显示 | RayNeo SDK 提供叠加层 API，或使用标准 `SurfaceView`/`TextureView` |
| 网络 | 标准 Android 网络 API，无限制 |

### 6.2 关键注意事项

- 眼镜端是 **Android 客户端**，使用标准 Android 网络 API 即可
- 若 RayNeo SDK 文档不可用，可退化为**标准 Android Camera2 + SurfaceView** 方案
- 网络请求必须在**非主线程**执行（Android 主线程禁止网络 IO）
- 眼镜 AR 显示分辨率较低，文字要大而简洁

## 7. 推荐技术选型

| 功能 | 推荐方案 | 备注 |
|------|----------|------|
| HTTP 客户端 | OkHttp 或 `HttpURLConnection` | OkHttp 更简洁 |
| JSON 解析 | `org.json.JSONObject` 或 Gson | 避免引入过多依赖 |
| 线程管理 | Kotlin 协程（`Dispatchers.IO`） | 或 `Thread` + `runOnUiThread` |
| 摄像头 | RayNeo Camera API（优先）或 Camera2 | 视 SDK 可用性决定 |
| AR 叠加显示 | RayNeo Display API（优先）或 OverlayView | 视 SDK 可用性决定 |

## 8. 参考代码片段

### 8.1 HTTP 请求（OkHttp + 协程）

```kotlin
import kotlinx.coroutines.*
import okhttp3.*
import okhttp3.MediaType.Companion.toMediaType
import okhttp3.RequestBody.Companion.toRequestBody
import org.json.JSONObject
import java.util.concurrent.TimeUnit

private val httpClient = OkHttpClient.Builder()
    .connectTimeout(5, TimeUnit.SECONDS)
    .readTimeout(10, TimeUnit.SECONDS)
    .build()

// 在协程中调用，例如 viewModelScope.launch { ... }
suspend fun sendImageToPhone(jpegBytes: ByteArray): DrugResult? {
    return withContext(Dispatchers.IO) {
        try {
            val body = jpegBytes.toRequestBody("image/jpeg".toMediaType())
            val request = Request.Builder()
                .url("http://192.168.43.1:8080/analyzeDrug")
                .post(body)
                .build()

            val response = httpClient.newCall(request).execute()
            if (response.isSuccessful) {
                val json = JSONObject(response.body!!.string())
                DrugResult(
                    drugName    = json.getString("drugName"),
                    warningText = json.getString("warningText"),
                    needConfirm = json.getBoolean("needConfirm"),
                    confidence  = json.getDouble("confidence")
                )
            } else null
        } catch (e: Exception) {
            null // 调用方检查 null 并显示错误提示
        }
    }
}

data class DrugResult(
    val drugName: String,
    val warningText: String,
    val needConfirm: Boolean,
    val confidence: Double
)
```

### 8.2 HTTP 请求（HttpURLConnection，无额外依赖）

```kotlin
import java.io.InputStream
import java.net.HttpURLConnection
import java.net.URL
import org.json.JSONObject

fun sendImageToPhone(jpegBytes: ByteArray): DrugResult? {
    return try {
        val conn = (URL("http://192.168.43.1:8080/analyzeDrug").openConnection()
                as HttpURLConnection).apply {
            requestMethod = "POST"
            doOutput = true
            setRequestProperty("Content-Type", "image/jpeg")
            setRequestProperty("Content-Length", jpegBytes.size.toString())
            connectTimeout = 5000
            readTimeout = 10000
        }

        conn.outputStream.use { it.write(jpegBytes) }

        if (conn.responseCode == 200) {
            val text = conn.inputStream.use { it.readBytes().toString(Charsets.UTF_8) }
            val json = JSONObject(text)
            DrugResult(
                drugName    = json.getString("drugName"),
                warningText = json.getString("warningText"),
                needConfirm = json.getBoolean("needConfirm"),
                confidence  = json.getDouble("confidence")
            )
        } else null
    } catch (e: Exception) {
        null
    }
}
```

## 10. UI 展示要求

AR 叠加界面需显示：

| 状态 | 显示内容 |
|------|----------|
| 空闲 | "点击拍照识别药物"提示 |
| 拍照中 | "正在识别..." |
| 成功 | `drugName` + `warningText`，若 `needConfirm=true` 则加"请确认"按钮 |
| 失败/超时 | "无法连接手机，请确认热点已开启" |

**字体大小**：AR 显示分辨率有限，建议标题 ≥ 24sp，正文 ≥ 18sp。

## 11. 错误处理

| 错误场景 | 判断方式 | 建议提示 |
|----------|----------|----------|
| 连接超时 | `catch (SocketTimeoutException)` | "无法连接手机，请确认热点已开启" |
| 网络不通 | `catch (IOException)` | "网络错误，请检查 WiFi 连接" |
| HTTP 500 | `responseCode == 500` | "手机识别服务出错，请重试" |
| `success: false` | JSON 字段判断 | 读取 `error` 字段显示或写日志 |
| 解析失败 | `catch (JSONException)` | "收到异常响应，请重试" |

## 12. 调试方法

### 12.1 验证手机端服务是否运行

在 PC 上执行（手机连接 USB 调试）：

```bash
adb shell "ss -tlnp | grep 8080"
```

有输出则说明手机端服务正在监听。

### 12.2 用 curl 模拟眼镜端

```bash
curl -X POST http://192.168.43.1:8080/analyzeDrug \
     -H "Content-Type: image/jpeg" \
     --data-binary @test.jpg
```

预期响应：

```json
{"success":true,"drugName":"测试药物","confidence":0.95,"warningText":"这是手机端返回的测试结果","needConfirm":true}
```

### 12.3 查看手机端日志（验证收到了眼镜请求）

```bash
adb logcat -s LocalHttpServer HttpServerService
```
