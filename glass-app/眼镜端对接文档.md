# 眼镜端对接文档

> 适用版本：手机端完成服务发现改造后
> 更新日期：2026-04-08

---

## 一、整体架构

```
眼镜端                              手机端（热点）
  │                                    │
  │  1. NSD Discover                   │  NsdManager 广播服务
  │ ─────────────────────────────────► │  serviceType: _rayneo-pill._tcp.
  │                                    │
  │  2. NSD Resolve → 得到 host:port   │
  │ ◄───────────────────────────────── │
  │                                    │
  │  3. HTTP POST /analyzeDrug         │  NanoHTTPD 监听 :8080
  │  Body: 图片二进制（image/jpeg）     │
  │ ─────────────────────────────────► │
  │                                    │
  │  4. HTTP 200 JSON 响应             │  调用大模型识别后返回
  │ ◄───────────────────────────────── │
```

眼镜端连接手机分两个阶段：
1. **服务发现阶段**：通过 DNS-SD/mDNS 找到手机的 IP 和端口，无需硬编码
2. **业务通信阶段**：拿到地址后，通过 HTTP 发送图片、接收识别结果

## 二、服务发现（NSD / DNS-SD）

### 2.1 关键参数

| 参数 | 值 |
|------|----|
| serviceType | `_rayneo-pill._tcp.` |
| serviceName | `rayneo_phone_bridge`（系统可能追加后缀，以 resolve 结果为准） |
| protocol | DNS-SD over mDNS（标准 Bonjour/Zeroconf） |
| 端口 | 由 resolve 结果动态获取，当前固定为 `8080` |

### 2.2 发现流程

```
discoverServices(serviceType)
    │
    ├─ onServiceFound(serviceInfo)
    │       └─ 过滤 serviceType == "_rayneo-pill._tcp."
    │
    └─ resolveService(serviceInfo)
            │
            └─ onServiceResolved(resolvedInfo)
                    ├─ host = resolvedInfo.host.hostAddress
                    ├─ port = resolvedInfo.port
                    └─ 保存地址，准备发起 HTTP 请求
```

### 2.3 Android 眼镜端示例代码

```kotlin
class PhoneServiceDiscovery(private val context: Context) {

    companion object {
        private const val TAG = "PhoneServiceDiscovery"
        private const val SERVICE_TYPE = "_rayneo-pill._tcp."
    }

    interface Listener {
        fun onPhoneFound(host: String, port: Int)
        fun onPhoneLost()
    }

    private val nsdManager = context.getSystemService(Context.NSD_SERVICE) as NsdManager
    private var discoveryListener: NsdManager.DiscoveryListener? = null
    private var listener: Listener? = null

    fun startDiscovery(listener: Listener) {
        this.listener = listener

        discoveryListener = object : NsdManager.DiscoveryListener {
            override fun onStartDiscoveryFailed(serviceType: String, errorCode: Int) {
                Log.e(TAG, "发现启动失败: errorCode=$errorCode")
            }

            override fun onStopDiscoveryFailed(serviceType: String, errorCode: Int) {
                Log.e(TAG, "发现停止失败: errorCode=$errorCode")
            }

            override fun onDiscoveryStarted(serviceType: String) {
                Log.i(TAG, "开始搜索服务: $serviceType")
            }

            override fun onDiscoveryStopped(serviceType: String) {
                Log.i(TAG, "停止搜索服务: $serviceType")
            }

            override fun onServiceFound(serviceInfo: NsdServiceInfo) {
                Log.i(TAG, "发现服务: ${serviceInfo.serviceName}")
                // 只处理目标服务类型
                if (serviceInfo.serviceType == SERVICE_TYPE) {
                    resolveService(serviceInfo)
                }
            }

            override fun onServiceLost(serviceInfo: NsdServiceInfo) {
                Log.w(TAG, "服务消失: ${serviceInfo.serviceName}")
                listener?.onPhoneLost()
            }
        }

        nsdManager.discoverServices(SERVICE_TYPE, NsdManager.PROTOCOL_DNS_SD, discoveryListener!!)
    }

    fun stopDiscovery() {
        discoveryListener?.let {
            try {
                nsdManager.stopServiceDiscovery(it)
            } catch (e: Exception) {
                Log.e(TAG, "停止发现异常", e)
            }
        }
        discoveryListener = null
    }

    private fun resolveService(serviceInfo: NsdServiceInfo) {
        nsdManager.resolveService(serviceInfo, object : NsdManager.ResolveListener {
            override fun onResolveFailed(serviceInfo: NsdServiceInfo, errorCode: Int) {
                Log.e(TAG, "Resolve 失败: errorCode=$errorCode")
            }

            override fun onServiceResolved(resolvedInfo: NsdServiceInfo) {
                val host = resolvedInfo.host.hostAddress ?: return
                val port = resolvedInfo.port
                Log.i(TAG, "Resolve 成功: host=$host, port=$port")
                listener?.onPhoneFound(host, port)
            }
        })
    }
}
```

### 2.4 注意事项

- **resolve 并发限制**：Android NsdManager 同一时刻只能有一个 resolve 在进行，若同时发现多个服务，需排队 resolve
- **服务消失处理**：`onServiceLost` 触发后，应清除缓存的 host/port，下次请求前重新 discover
- **热点重启场景**：手机热点重启后，眼镜需重新执行 discover → resolve 流程，不要复用旧地址
- **Android 13+ 变更**：API 33 起 `NsdManager` 新增了基于 executor 的异步 API，建议在高版本上使用新 API 避免回调线程问题

## 三、HTTP 业务接口

### 3.1 接口概览

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/analyzeDrug` | 发送药物图片，获取识别结果 |

Base URL 格式：`http://<resolved_host>:<resolved_port>`

---

### 3.2 POST /analyzeDrug

#### 请求

| 项目 | 值 |
|------|----|
| Method | `POST` |
| Content-Type | `image/jpeg` |
| Body | 图片原始二进制数据（不要 Base64，不要 multipart） |

```
POST http://192.168.43.1:8080/analyzeDrug
Content-Type: image/jpeg
Content-Length: <图片字节数>

<图片二进制数据>
```

#### 响应

HTTP 200，Content-Type: `application/json`

```json
{
  "success": true,
  "drugName": "阿莫西林胶囊",
  "confidence": 0.95,
  "warningText": "请按医嘱服用，勿超量",
  "needConfirm": false
}
```

#### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `success` | Boolean | 是否识别成功 |
| `drugName` | String | 识别出的药物名称 |
| `confidence` | Double | 置信度，范围 0.0 ~ 1.0 |
| `warningText` | String | 给用户的提示或警告文本 |
| `needConfirm` | Boolean | 是否需要用户二次确认（如存在剂量风险） |

#### 错误响应

| HTTP 状态码 | 场景 |
|-------------|------|
| 404 | 访问了不存在的路径 |
| 500 | 服务器内部错误（如图片读取失败） |

错误响应体格式：
```json
{
  "success": false,
  "error": "错误描述"
}
```

### 3.3 眼镜端请求示例代码

```kotlin
suspend fun analyzeDrug(host: String, port: Int, imageBytes: ByteArray): DrugAnalyzeResponse {
    val url = URL("http://$host:$port/analyzeDrug")
    val connection = (url.openConnection() as HttpURLConnection).apply {
        requestMethod = "POST"
        setRequestProperty("Content-Type", "image/jpeg")
        setRequestProperty("Content-Length", imageBytes.size.toString())
        doOutput = true
        connectTimeout = 5_000
        readTimeout = 30_000  // 大模型推理可能耗时较长
    }

    connection.outputStream.use { it.write(imageBytes) }

    val responseCode = connection.responseCode
    if (responseCode != HttpURLConnection.HTTP_OK) {
        throw IOException("HTTP 错误: $responseCode")
    }

    val responseJson = connection.inputStream.bufferedReader().readText()
    return Gson().fromJson(responseJson, DrugAnalyzeResponse::class.java)
}
```

## 四、完整连接流程

```
应用启动
    │
    ▼
startDiscovery()
    │
    ├─ 未发现服务 → 等待，持续搜索
    │
    └─ onPhoneFound(host, port)
            │
            ▼
        缓存 host + port
            │
            ▼
        用户触发拍照
            │
            ▼
        POST /analyzeDrug (imageBytes)
            │
            ├─ 成功 → 展示识别结果
            │           needConfirm=true → 弹出二次确认
            │
            └─ 失败（连接超时/拒绝）
                    │
                    └─ 清除缓存地址
                       重新 discover → resolve
                       重试请求
```

## 五、调试建议

### 5.1 验证手机端服务已发布

在 Android Studio Logcat 中过滤 tag：

```
adb logcat -s NsdServicePublisher PhoneBridgeController LocalHttpServerManager
```

看到以下日志说明手机端正常：
```
NsdServicePublisher: NSD 注册成功: serviceName=rayneo_phone_bridge, serviceType=_rayneo-pill._tcp., port=8080
```

### 5.2 验证眼镜端能发现服务

在眼镜端 Logcat 中过滤 tag：
```
adb logcat -s PhoneServiceDiscovery
```

期望看到：
```
PhoneServiceDiscovery: Resolve 成功: host=192.168.43.1, port=8080
```

### 5.3 直接测试 HTTP 接口

手机和眼镜连接同一热点后，可用 curl 验证接口可达：

```bash
curl -X POST http://192.168.43.1:8080/analyzeDrug \
  -H "Content-Type: image/jpeg" \
  --data-binary @test.jpg
```

## 六、常见问题

**Q: discover 一直没有回调？**
- 确认眼镜和手机在同一局域网（眼镜连接手机热点）
- 确认手机端服务已启动（Logcat 有 NSD 注册成功日志）
- 部分设备 mDNS 组播被防火墙拦截，尝试重启热点

**Q: resolve 失败，errorCode=3？**
- errorCode=3 (`FAILURE_ALREADY_ACTIVE`) 表示有另一个 resolve 正在进行，需排队等待后重试

**Q: HTTP 请求超时？**
- 确认端口 8080 未被其他应用占用
- 确认手机端 HttpServerService 前台服务正在运行（通知栏有持久通知）

**Q: `needConfirm=true` 时应如何处理？**
- 展示 `warningText` 内容，要求用户明确确认后再执行后续操作（如取药、服药）
