# MediView 系统使用说明书

> 适用版本：包含服药提醒推送功能的最新版本

## 一、系统是什么

MediView 是一套"手机 + AR 眼镜"配合使用的服药辅助系统，由两个 App 组成：

- **手机端 App**（`phone_app`）：作为整个系统的"大脑"和服务器，负责设置提醒、管理药物信息、对眼镜拍到的药物图片做识别
- **眼镜端 App**（`glass-app`）：运行在 Rayneo（雷鸟）AR 眼镜上，负责拍照识别药物、显示服药提醒

两台设备之间不需要联网（不需要外网），通过**手机开热点、眼镜连上手机热点**的局域网方式互通。

系统目前提供两大功能：

1. **药物识别**：眼镜拍照 → 手机识别 → 眼镜显示药品名、用法、用量、禁忌
2. **服药提醒**：手机上设好提醒 → 到时间 → 眼镜上弹出提醒画面 → 用户做手势确认

## 二、准备工作

### 2.1 你需要什么

- 一部 Android 手机（**Android 12 及以上**）
- 一副雷鸟（Rayneo）AR 眼镜（Mercury 平台）
- 两个 App 都已经安装到对应设备
- 手机已经能开启个人热点

### 2.2 第一次使用前的设置

1. **打开手机的个人热点**
   - 进入手机系统设置 → 个人热点（或称"便携式热点"），打开开关
   - 记下热点名称，以便眼镜连接
2. **眼镜连上手机的热点**
   - 在眼镜的 WiFi 设置里选择刚才打开的手机热点，输入密码连上
3. **同意手机端必要的权限**
   - 首次打开手机端 App 时，系统会请求"发送通知"等权限，全部允许
   - **重要**：到手机系统设置 → 应用管理 → 找到 MediView → "闹钟和提醒"权限，确保开启。否则到点闹钟无法精确触发

## 三、手机端使用

打开手机端 App，底部有三个标签页：

```
┌─────────┬────────────┬──────────┐
│  服务   │  服药提醒  │ 网络设置 │
└─────────┴────────────┴──────────┘
```

### 3.1 服务页

- 这一页是控制后台服务的开关
- 点击"启动服务"按钮，状态栏会出现一条常驻通知，表示后台 HTTP 服务在跑（眼镜要找的就是这个服务）
- 通常**让它一直开着即可**。如果服务没启动，眼镜不会发现手机、提醒也送不过去

### 3.2 服药提醒页

#### 添加一条提醒

1. 点右下角的"+"加号按钮
2. 在弹出的窗口中填写：
   - **药品名称**：例如"二甲双胍"
   - **每日次数**：例如 3
   - **每次用量**：例如 1
   - **单位**：从「粒、毫升、支、片」中选
   - **提醒时间**：可以添加多个时间，例如 08:00、12:00、19:00
   - **重复周期**：选哪几天提醒，默认是每天
3. 点"确认"保存

#### 管理已有提醒

- 每条提醒都有一个开关：**关掉**表示暂停提醒（不会再到点触发），**打开**才会真正提醒
- 点编辑图标修改信息；点删除图标移除

#### 提醒到了会怎样

- 到达设定时间后，**手机本身不会弹通知**——这是有意为之，因为提醒是给眼镜看的
- 这条提醒会进入"待推送队列"，等眼镜来取
- 眼镜每 30 秒查一次手机，**最迟 30 秒内**眼镜就会弹出提醒画面

### 3.3 网络设置页

- 可以查看手机当前的 IP 地址，便于调试
- 一般不需要改动

## 四、眼镜端使用

戴上眼镜，启动 MediView 眼镜 App，会直接进入**拍照识别页**。

### 4.1 手势说明

整个眼镜端用两个手势操作（在眼镜镜腿上轻点）：

| 手势 | 在拍照页 | 在提醒页 |
|------|----------|----------|
| 单击 | 拍照识别 | 确认服药 |
| 双击 | 退出 App | 稍后再提醒 |

### 4.2 药物识别

1. 进入拍照页后，眼镜会自动搜索手机服务
2. 看到提示"已连接手机服务"后即可使用
3. **对准药品** → **单击** → 等几秒
4. 眼镜底部会出现一张半透明卡片，显示：
   - 药品名
   - 用法（如"温水送服"）
   - 用量（如"一日三次"）
   - 禁忌（黄色字，需要注意的事项）

如果一直显示"正在搜索手机服务..."，看下面的"常见问题"。

### 4.3 服药提醒

- 你**不需要操作**任何东西。眼镜在后台自动轮询手机
- 一旦手机推送了提醒，眼镜屏幕会**自动切换**到提醒画面：

```
        ┌────────────────────────┐
        │     服药提醒（黄色）     │
        │                        │
        │       08:00            │   ← 大字号的时间
        │                        │
        │     二甲双胍             │   ← 药品名
        │  一天 3 次 每次 1 粒     │   ← 剂量
        │                        │
        │    单击确认 双击稍后    │
        └────────────────────────┘
```

- **单击**：表示"我已经服药了" → 提醒消失，回到拍照页；同时手机端会把这条提醒移除
- **双击**：表示"等会儿再说" → 提醒消失但不算确认；如果一直不确认，下次重启 App 时还会再弹出

## 五、典型使用流程示例

**场景：王阿姨需要每天三餐前吃一种降压药。**

1. **王阿姨的家人**在手机端 App 上：
   - 打开"服务"页，确认服务已启动
   - 进入"服药提醒"页，添加一条提醒：
     - 名称：硝苯地平
     - 一天 3 次，每次 1 片
     - 时间：07:30、11:30、17:30
     - 重复：每天
2. **王阿姨**戴上眼镜，眼镜进入拍照页（默认）
3. 到了 **07:30**：
   - 手机端闹钟触发，把提醒放进队列
   - 大约 30 秒内，王阿姨眼镜上的画面切换到提醒页，看到"07:30 硝苯地平 一天 3 次 每次 1 片"
   - 王阿姨吃完药，**轻点眼镜镜腿一下**确认
   - 眼镜回到拍照页
4. 11:30 和 17:30 同样会自动提醒

## 六、常见问题

### Q1：眼镜端一直显示"正在搜索手机服务..."

检查清单：

- [ ] 手机的个人热点是不是打开了？
- [ ] 眼镜是不是连上了**手机的**热点（不是别的 WiFi）？
- [ ] 手机端 App 是不是在"服务"页点了"启动服务"？状态栏应该有一条常驻通知
- [ ] 手机端 App 是不是被系统杀了？建议在系统设置里给 MediView 加上"自启动 / 不优化电池"

### Q2：到了提醒时间，眼镜没有弹出提醒

检查清单：

- [ ] 手机端这条提醒的开关是不是处于**打开**状态？
- [ ] 时间和重复日是不是设对了？例如只勾选了"周一"则其他日子不会提醒
- [ ] 手机系统设置里 MediView 的"闹钟和提醒"权限是不是开了？
- [ ] 眼镜和手机的网络通不通？回到 Q1 检查
- [ ] 等够了吗？最多需要 30 秒延迟

### Q3：眼镜端反复弹出同一条提醒

- 单击确认后会真正消除；如果只是双击"稍后"，提醒在 30 分钟内仍可能再次弹出
- 如果手机端这条提醒的时间到了，且 30 分钟内一直没被确认，下次眼镜重新连上时还会弹

### Q4：提醒画面信息显示不全 / 显示错位

- 这是 AR 眼镜的双目同步显示，正常情况下两只眼睛看到的内容一致
- 如果有错位，重启眼镜 App（双击退出再打开）

### Q5：手机重启之后提醒还能用吗

- 当前版本：手机重启后**需要重新打开一次手机端 App**，并进入一次"服药提醒"页，所有提醒才会重新注册到系统闹钟
- 平时正常使用不需要操作（CRUD 操作会自动重排）

### Q6：药物识别识别不准确

- 当前版本如果没有配置真实的大模型，会返回模拟数据
- 真实识别需要在「网络设置」页配置可用的大模型 API

## 七、注意事项

1. **手机端 App 需要长时间在后台运行**：建议给它"无限制电池使用"权限，否则系统可能在睡眠时杀掉 HTTP 服务，导致眼镜断连
2. **手机和眼镜在同一个局域网**才能通信：典型场景是手机开热点、眼镜连热点
3. **首次使用务必允许"闹钟和提醒"权限**：否则定时无法精确触发，可能延迟数分钟到十几分钟
4. **服药提醒不能替代医生医嘱**：本系统只是辅助提醒工具，请遵医嘱用药
